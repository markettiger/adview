# 광고 데이터 대시보드 (PC)

메타 애즈 / 애플 애즈 데이터를 한 화면에서 확인하는 정적 대시보드입니다.

## 변경된 핵심 UX

- API 연동 UI는 상단 고정이 아니라 **왼쪽 사이드바의 `API 연동 관리` 페이지**로 이동했습니다.
- 대시보드의 기간 버튼은 **어제 / 오늘 / 지난주 / 이번주 / 직접 선택(달력)** 으로 구성됩니다.

## 요구사항 반영

1. **메타/애플 API 연동 지원 (연동 관리 페이지로 분리)**
   - `API Base URL` + 각 플랫폼 `광고 ID`를 저장합니다.
   - 대시보드의 `저장된 연동으로 데이터 새로고침` 버튼으로 실제 API를 조회합니다.
   - 연동 해제 버튼으로 저장된 설정을 제거할 수 있습니다.

2. **요약 KPI + 표 + 그래프 + 날짜 필터**
   - 기간 필터: `어제`, `오늘`, `지난주`, `이번주`, `직접 선택(시작/종료일)`
   - 상단 KPI 카드에 총 노출/클릭/전환/비용 합계가 표시됩니다.
   - 표에는 주요 지표(노출/클릭/전환/비용/CTR)가 표시됩니다.
   - 그래프는 선택 기간의 Impressions 추이를 보여줍니다.

3. **왼쪽 사이드바 분리 보기**
   - 전체 / 메타 / 애플 데이터 필터를 제공합니다.

## API 엔드포인트

- `GET /api/meta-ads?adId={META_AD_ID}`
- `GET /api/apple-ads?adId={APPLE_AD_ID}`

## API 응답 형식 예시

```json
{
  "data": [
    {
      "date": "2026-04-21",
      "impressions": 12000,
      "clicks": 580,
      "conversions": 33,
      "spend": 320000
    }
  ]
}
```

## 실행

브라우저에서 `index.html`을 열면 됩니다.

- 실제 API를 연결하지 않아도, `샘플 데이터 보기` 버튼으로 UI 검증 가능
- CORS 정책이 있는 경우 API 서버에서 허용 설정 필요

## PR 반영 체크

- 브랜치 최신 커밋이 반영되지 않은 경우, 변경사항 커밋 후 PR을 다시 생성해 동기화할 수 있습니다.
- 본 저장소는 현재 `API 연동 관리` 페이지 분리와 날짜 필터(어제/오늘/지난주/이번주/직접 선택) 요구사항이 반영된 상태입니다.

## 브랜치/PR 작업 기준

- 본 변경은 최신 `main` 기반 새 브랜치에서 작업하는 것을 원칙으로 합니다.
- PR 생성 시 base는 `main`으로 지정해 병합되도록 진행합니다.
- 핵심 변경 파일(`index.html`, `app.js`, `styles.css`)이 `Files changed`에 표시되는지 확인합니다.


## PR 전 체크

- PR 생성 전 `git diff --name-only main...HEAD`로 최소 4개 파일 변경 여부를 확인합니다.
- `index.html`, `app.js`, `styles.css`, `README.md`가 반드시 변경 목록에 포함되어야 합니다.
